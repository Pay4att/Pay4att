## Design Context

### Users
This profile is for visitors browsing a personal GitHub presence rather than a product site. The main audience is people evaluating the owner through repositories and public work: peers, potential collaborators, and anyone curious about ongoing experiments.

### Brand Personality
Minimal, precise, and sharper in presence. The profile should still feel premium, but it now needs visible motion and live GitHub signals so the page feels active rather than static.

### Aesthetic Direction
Dynamic editorial minimalism inside GitHub's README constraints. Use restrained motion surfaces such as typing headers, live stats, contribution animation, and activity graphs, but keep the composition clean and avoid turning the page into a random badge dump.

### Design Principles
1. Lead with one strong visual gesture, then let the content breathe.
2. Keep copy short and specific; do not invent credentials or experience that are not publicly grounded.
3. Prefer a small number of dynamic modules over static decoration.
4. Make the README work inside GitHub's rendering constraints without feeling template-driven.
5. Build for longevity: easy to update, readable in both light and dark themes, and still effective with a small public repo set.
